# ViWallet v2.1 — Design System Overhaul (Design Spec)

**Date:** 2026-06-25
**Status:** Approved for planning
**Scope:** Visual redesign + light/dark theming + animation system. **No new features, no data-model changes.**

---

## 1. Goal

Transform ViWallet's current dark-glassmorphism look into a soft, modern, light-first fintech aesthetic (reference: rounded white cards on a lavender canvas, a dark "analytics" hero card with a pastel donut, pill toggles, large bold numerals). Add a complete light/dark theming engine and a reusable animation layer. This is the foundation that all later feature work (v2.2) is built on, so every new feature inherits the new look and motion for free.

## 2. Non-Goals

- No new product features (savings goals, debt, transfers, etc. — those are v2.2).
- No changes to the Prisma schema, API routes, or data model.
- No changes to auth, deployment, or business logic.
- Purely presentation, theming, and motion.

## 3. Theming Engine

**Library:** `next-themes` (already installed, v0.4.6).

- `ThemeProvider` wraps the app in `app/layout.tsx` with `attribute="class"`, `defaultTheme="light"`, `enableSystem`, and `disableTransitionOnChange` to avoid flash.
- Adds/removes `.dark` on `<html>`. Persists choice in `localStorage`; respects system preference until the user explicitly chooses.
- **Tailwind v4 CSS-first** (no `tailwind.config.ts`). In `app/globals.css`:
  - Define a dark variant: `@custom-variant dark (&:where(.dark, .dark *));`
  - Declare all color tokens as CSS variables in `:root` (light) and override under `.dark`.
  - Expose tokens through the `@theme` block so utilities (`bg-surface`, `text-secondary`, `text-primary`, `bg-primary`, etc.) resolve in both modes.
- Prevent FOUC: `suppressHydrationWarning` on `<html>`; theme script injected by `next-themes`.

### 4.2 Typography

- **Family:** Plus Jakarta Sans (Google Fonts), weights 400, 500, 600, 700, 800. Replaces IBM Plex Sans everywhere.
- **Scale:** display/balance 34px/800 (tracking −1px); h1 22px/700; h2 18px/700; body 14–16px/400–500; label 11–13px/500; secondary 11px/500.
- Tabular figures for monetary values.

### 4.3 Shape & Depth

- **Radii:** `sm` 12px, `md` 18px, `lg` 24px, `pill` 999px. Cards use `lg`; icon tiles use `md`.
- **Shadows (light only):** card `0 6px 16px rgba(86,72,150,.08)`; elevated/FAB `0 18px 50px rgba(86,72,150,.18)` / `0 10px 24px rgba(124,111,232,.45)`.
- **Dark mode:** shadows disabled; depth via surface contrast + hairline `border` token.
- **Spacing rhythm:** 4 / 8 / 12 / 16 / 20 / 28px.

## 5. Animation System (Motion / Framer Motion)

**Library:** `motion` (install). All motion components are client components and live in `components/motion/`. Everything respects `prefers-reduced-motion` (falls back to instant/opacity-only).

| Primitive | Component | Behavior |
|-----------|-----------|----------|
| Page transition | `app/(app)/template.tsx` | Fade + 8px upward slide on route change |
| Animated number | `AnimatedNumber` | Counts up on mount and on value change (spring); used for balances/totals |
| Bottom sheet / modal | `Sheet` upgrade | Spring slide-up, backdrop fade, drag-to-dismiss on mobile |
| List entrance | `Stagger` / `motion.li` | Staggered fade-in-up for transaction rows and card grids |
| Tap/hover feedback | `Pressable` wrapper | `whileTap` scale 0.97; hover lift (light mode) on cards/buttons |
| Donut & progress | within chart/`BudgetCard` | Animated arc draw / bar fill on mount |
| Skeletons | `Skeleton` | Shimmer placeholders during data fetch (replace spinners) |
| Theme switch | `ThemeToggle` | Icon crossfade/rotate sun↔moon |

## 6. Component Inventory

### 6.1 New components
- `components/theme/ThemeProvider.tsx` — wraps next-themes.
- `components/theme/ThemeToggle.tsx` — sun/moon toggle; placed in Settings and in the app header/sidebar.
- `components/motion/AnimatedNumber.tsx`

- `components/motion/Pressable.tsx` — tap/hover wrapper.

- `components/shared/IconTile.tsx` — rounded tinted icon backer (category color → tint).

### 6.2 Restyled components
Card, Button, AmountInput, pill Toggle (income/expense), Sidebar, BottomNav, Modal/Sheet, BudgetCard, TransactionRow, SpendingDonut, IncomeExpenseArea, EmptyState. All move to the new tokens, radii, shadows, and IconTile usage.

## 7. Page Re-skin Notes

All existing pages keep their data flows and routes; only presentation changes.

- **Login** — lavender canvas, centered white card, large logo, single primary "Continue with Google" button with tap feedback.
- **Onboarding** — soft card, animated step transitions, currency picker restyled.
- **Dashboard** — total-balance header with `AnimatedNumber` + trend pill; dark hero analytics card with animated donut + legend; quick-stat cards driven **only by existing data** (e.g. top budget progress, this-month spend) — no savings-goal card (goals are v2.2); recent transactions with staggered entrance.
- **Transactions** — pill income/expense toggle, weekly spending bar mini-chart, transaction rows with IconTile, circular primary FAB, spring add-sheet.
- **Budgets** — restyled BudgetCard with animated progress fill; amber/red threshold colors mapped to new palette.
- **Trips** — list cards + detail with animated totals and staggered transaction list.
- **Reports** — donut + area chart on new palette with mount animations; date-range control restyled.
- **Settings** — grouped cards; ThemeToggle row (Light / Dark / System); existing wallet/category links restyled.

## 8. Accessibility & Non-Functional

- WCAG AA contrast (≥4.5:1) verified for both light and dark token pairs.
- `prefers-reduced-motion` honored by every animation primitive.
- No layout shift from theming (CSS variables, no JS reflow); no FOUC.
- Keyboard focus rings on all interactive elements in both themes.
- Responsive breakpoints unchanged: 375 / 768 / 1024 / 1440px; bottom nav ≤768px, sidebar ≥1024px.
- `npm run build` passes; existing tests remain green (presentation changes only).

## 9. Acceptance Criteria

1. Toggling Light/Dark/System restyles the entire app instantly with no flash, and the choice persists across reloads.
2. Every page matches the soft-modern reference: lavender/dark canvas, rounded white/`surface` cards, IconTiles, pill toggles, large bold numerals, dark hero analytics card.
3. Plus Jakarta Sans is the active font app-wide.
4. Balances/totals animate (count-up); route changes fade+slide; the add-transaction sheet springs up; lists stagger in; donut/progress animate on mount; skeletons replace spinners — all disabled under reduced-motion.
5. No feature behavior or data changed; build is green.
