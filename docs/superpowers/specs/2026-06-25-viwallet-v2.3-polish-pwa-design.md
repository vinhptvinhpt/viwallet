# ViWallet v2.3 — Polish & PWA (Design Spec)

**Date:** 2026-06-25
**Status:** Approved for planning
**Depends on:** v2.1 (design system) and v2.2 (features) shipped.
**Scope:** Last-mile shine — micro-interactions, a cohesive empty-state system, haptic-style feedback, a performance pass, and PWA hardening. **No new product features.**

---

## 1. Goal

Make ViWallet feel finished and installable: delightful micro-interactions, consistent empty states, tactile feedback on mobile, fast loads, and a proper installable PWA with offline resilience and update prompts.

## 2. Non-Goals

- No new product features or data-model changes.
- No visual-language change (v2.1 tokens stay).
- Web Push for bill reminders is an **optional stretch** at the end; the default reminder path remains in-app + local Notification API from v2.2.

## 3. Workstreams

### 3.1 Micro-interactions
- Success feedback: animated checkmark on save (transaction, goal contribution, debt payment).
- Goal completion celebration: lightweight confetti burst when a goal hits 100% (respects reduced-motion → static badge).
- Button/press feedback everywhere via the v2.1 `Pressable` (audit any element still missing it).
- Number ticks: ensure all balances/totals use `AnimatedNumber` (audit pages added in v2.2).
- Tab/route transitions consistent across all routes including new Goals/Debts.

### 3.2 Empty-state system
- One reusable `EmptyState` component variant set with a tinted `IconTile`, headline, subtext, and a primary action.
- Applied consistently: no wallets, no transactions, no budgets, no trips, no goals, no debts, no search results, no upcoming bills, empty import preview.

### 3.3 Haptic-style feedback (mobile)
- `navigator.vibrate` wrapper fired on key actions (save, delete, toggle, goal complete). Feature-detected; no-op where unsupported; gated by a user setting (default on).

### 3.4 Performance pass
- Dynamic-import heavy client components (Recharts charts, ImportWizard, lightbox) so they don't bloat first load.
- Verify route-level code splitting; lazy-load below-the-fold dashboard sections.
- Image optimization for receipts/trip covers (`next/image` where applicable; sized thumbnails).
- Target Lighthouse ≥90 Performance and ≥95 PWA on the dashboard (mobile profile). Record before/after.

### 3.5 PWA hardening
- Real icon set: 192, 512, and maskable variants; correct `manifest.json` (name, theme/background per v2.1 light + dark), `apple-touch-icon`.
- **Install prompt UI:** capture `beforeinstallprompt`, show a dismissible "Install ViWallet" banner; respect dismissal.
- **Offline fallback:** service worker serves a branded offline page for navigations when offline (extends the existing `public/sw.js` from v1).
- **Update flow:** detect a new service worker and show an "Update available — reload" toast.
- Keep the existing offline transaction queue (v1) working; verify it flushes on reconnect.
- **Optional stretch:** Web Push for bill reminders (VAPID keys, push subscription stored server-side, push on due date). Clearly optional; only if time permits.

## 4. Non-Functional

- Every animation respects `prefers-reduced-motion`.
- Haptics and install prompt are dismissible and remember the user's choice.
- No regression to v2.1/v2.2 behavior; `npm run build` green; existing tests pass.
- PWA installable on Android Chrome and desktop Chrome; offline navigation shows the fallback, online restores normally.

## 5. Acceptance Criteria

1. Saving a transaction/contribution/payment shows a success checkmark; completing a goal triggers confetti (static under reduced-motion).
2. Every list shows a consistent, branded empty state with a clear primary action.
3. Key actions produce a haptic tick on supporting mobile devices; the setting toggles it off.
4. Heavy components are dynamically imported; dashboard Lighthouse Performance ≥90 and PWA ≥95 (mobile), with before/after recorded.
5. The app is installable with proper icons; an install banner appears and is dismissible; an update toast appears when a new SW is available; offline navigation shows the branded fallback and the v1 offline queue still flushes on reconnect.
6. Build is green; no feature/behavior regressions.
