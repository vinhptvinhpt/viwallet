# ViWallet v2.1 — Design System Overhaul Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Re-skin ViWallet into a soft, modern, light-first fintech aesthetic with a full light/dark theming engine and a reusable Motion-based animation layer — no feature or data-model changes.

**Architecture:** `next-themes` drives a `.dark` class on `<html>`; all colors are CSS variables (light in `:root`, dark under `.dark`) exposed through Tailwind v4's `@theme` block. A small set of reusable client components in `components/motion/` and `components/shared/` provide animation and presentation primitives that every page (and all future v2.2 features) consume.

**Tech Stack:** Next.js 16 (App Router), TypeScript strict, Tailwind v4 (CSS-first), `next-themes`, `motion` (Framer Motion), Plus Jakarta Sans, Lucide React, Recharts, Jest + Testing Library (jsdom).

## Global Constraints

- Next.js 16 App Router; TypeScript strict mode.
- Theming: `next-themes`, `attribute="class"`, `defaultTheme="light"`, `enableSystem`, `disableTransitionOnChange`. Tailwind v4 CSS-first — NO `tailwind.config.ts`. Dark variant via `@custom-variant dark (&:where(.dark, .dark *))`.
- Font: Plus Jakarta Sans (Google Fonts), weights 400/500/600/700/800. Replaces IBM Plex Sans everywhere.
- Color tokens (light / dark): `canvas` #F3F1FA / #0E0D12 · `surface` #FFFFFF / #1C1B22 · `surface-2` #FBFAFE / #26242E · `hero` #1C1B22 / #000000 · `text-primary` #1A1825 / #F5F4F8 · `text-secondary` #9B98A8 / #9B98A8 · `primary` #7C6FE8 / #8B7FF0 · `primary-soft` #EAE7FA / #2A2740 · `success` #10936A / #34D39E · `danger` #E24B4A / #F0726E. Categories (both modes): mint #5DD6A8, purple #7C6FE8, amber #FBC54D, coral #F08A7C.
- Radii: sm 12px, md 18px, lg 24px, pill 999px. Cards use lg, icon tiles md.
- Shadows light only: card `0 6px 16px rgba(86,72,150,.08)`, elevated `0 18px 50px rgba(86,72,150,.18)`. Dark mode: no shadows; hairline border `rgba(255,255,255,.08)`.
- Every animation respects `prefers-reduced-motion` (opacity-only / instant fallback).
- WCAG AA contrast (≥4.5:1) in both themes. Keyboard focus rings in both themes.
- Breakpoints unchanged: 375 / 768 / 1024 / 1440px; bottom nav ≤768px, sidebar ≥1024px.
- Amounts remain integer cents; divide by 100 for display. No data-model/API changes.
- **Testing approach:** Component behavior (theme class toggling, render output, reduced-motion branch) is tested with Testing Library + jsdom. Pure helpers (e.g. `categoryTint`, number formatting) get node unit tests. CSS/visual correctness (token resolution, Tailwind utilities, layout, animation rendering) is verified by `npm run build` plus the per-task manual visual checklist — do not write assertions that pretend to verify rendered pixels.
- After every task: `npm run build` must pass and existing tests stay green.

```ts
function hexToRgb(hex: string): [number, number, number] | null {
  const m = hex.replace('#', '').match(/^([0-9a-f]{6})$/i)
  if (!m) return null
  const n = parseInt(m[1], 16)
  return [(n >> 16) & 255, (n >> 8) & 255, n & 255]
}

export function categoryTint(color: string): { bg: string; fg: string } {
  const fallback = '#7C6FE8'
  const rgb = hexToRgb(color || fallback)
  const [r, g, b] = rgb ?? hexToRgb(fallback)!
  return {
    fg: color || fallback,
    bg: `rgba(${r}, ${g}, ${b}, 0.16)`,
  }
}
```

`components/motion/AnimatedNumber.tsx`:
```tsx
'use client'

import { useEffect, useRef, useState } from 'react'
import { animate } from 'motion'
import { useAppReducedMotion } from './useReducedMotion'

export default function AnimatedNumber({
  value,
  format = (n: number) => String(Math.round(n)),
  className,
}: {
  value: number
  format?: (n: number) => string
  className?: string
}) {
  const reduced = useAppReducedMotion()
  const [display, setDisplay] = useState(value)
  const prev = useRef(value)

  useEffect(() => {
    if (reduced) {
      setDisplay(value)
      prev.current = value
      return
    }
    const controls = animate(prev.current, value, {
      duration: 0.8,
      ease: 'easeOut',
      onUpdate: (v) => setDisplay(v),
    })
    prev.current = value
    return () => controls.stop()
  }, [value, reduced])

  return <span className={className}>{format(display)}</span>
}
```
(On first render `display === value`, so the test sees the final formatted value synchronously.)
